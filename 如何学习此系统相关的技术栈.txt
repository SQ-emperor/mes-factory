 项目经验：厂里通 — 轻量级制造执行系统 (MES)

  技术栈：Next.js 16 (App Router) + TypeScript + Prisma 7 + Neon PostgreSQL + Tailwind CSS + Base UI

  项目描述：独立设计开发的面向中小工厂的轻量级 MES 系统，支持手机扫码报工、订单全生命周期管理、AI
  智能排产和生产质量管理。

  核心功能：
  - 认证与权限：基于 NextAuth v5 的手机验证码登录，JWT 会话管理，admin/manager/worker 三级角色层级权限控制
  - 数据看板：Recharts 可视化图表展示生产进度、订单状态、质量概况
  - 订单管理：订单创建、状态流转（待排产→已排产→生产中→已完成）、工单拆分
  - AI 排产：基于工序工时和设备资源的自动排产算法，预估订单完成时间
  - 扫码报工：支持订单/工位二维码扫码、报工提交、HTML5-QRCode 浏览器扫码
  - 质量管理：来料检、过程检、成品检，支持缺陷记录和图片上传
  - PWA 离线支持：Service Worker 缓存策略，离线报工数据本地存储(Dexie IndexedDB)，联网自动同步
  - 响应式设计：PC 端侧边栏 + 手机端底部导航栏自适应

  个人贡献：独立完成全栈开发 — 数据库 Schema 设计、Next.js 前后端架构、Server Actions 接口、UI 组件封装、PWA
  离线方案、AI 排产算法。

  面试高频问题 & 回答模板

  Q1: 为什么选 Next.js 而不是 Vue/传统 React？

  ▎ 项目需要手机扫码+离线使用，Next.js 自带 PWA 支持和 Server Actions，可以在一套代码里完成前后端，不用单独写 API 服务。

  Q2: 数据库是怎么设计的？

  ▎ 多租户架构，一个工厂一个 Tenant，下面关联 User、Order(订单)、Product(产品)、Process(工序)等表，用 Prisma
  ▎ 管理表结构和迁移。

  Q3: 权限怎么做的？

  ▎ admin > manager > worker 三级角色层级，JWT token 里携带角色信息，Server Actions 里用 canManage()
  ▎ 函数校验上级才能操作下级。

  Q4: AI 排产是什么？

  ▎ 不是真 AI，是基于贪心算法的自动排产——按订单优先级和工序工时，把工单分配到设备和时间段，计算预计完成时间。
